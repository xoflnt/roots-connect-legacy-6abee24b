

# بوابة تراث آل الخنيني — خارطة التطوير الشاملة

---

## تحليل الوضع الحالي

**ما تم إنجازه:** بوابة متعددة الواجهات (شجرة تفاعلية، سلسلة نسب، قائمة متداخلة، جدول بيانات) مع صفحة هبوط فاخرة، دعم الوضع الداكن/الفاتح، وبيانات 5+ أجيال في ملف ثابت.

**نقاط القوة:** تجربة RTL أصيلة، ألوان تراثية مميزة، تصميم متجاوب، أربع طرق عرض تخدم مستخدمين مختلفين.

**النقاط العمياء الحالية:**
1. **البيانات مسطحة**: `spouses` هو نص حر (string) وليس علاقة حقيقية — لا يمكن ربط الأبناء بأم محددة برمجياً
2. **لا يوجد Deep Linking**: لا يمكن مشاركة رابط مباشر لشخص معين (كل شيء على `/`)
3. **لا يوجد حالة "متوفى صغيراً"**: المعلومة موجودة في `notes` كنص حر بدلاً من حقل مهيكل
4. **لا يوجد ترتيب للأبناء**: لا طريقة لمعرفة الأكبر من الأصغر إلا بالتاريخ (وليس دائماً متوفر)

## الركيزة 1: تجربة عاطفية وشخصية

### أ. حاسبة صلة القرابة
- المستخدم يختار شخصين → النظام يحسب المسار بينهما ويعرض العلاقة بالعربية ("ابن عمك الثاني" أو "جدك لأبيك")
- عرض المسار بصرياً كسلسلة مصغرة

### ب. شارات تراثية (Heritage Badges)
- **المؤسس**: لناصر سعدون الخنيني
- **أب الفرع**: لكل من له 5+ أبناء
- **الجيل الأول/الثاني/...**: شارة الجيل على كل بطاقة
- **رحمه الله**: شارة خاصة للمتوفين بتصميم وقور

### ج. صفحة "بطاقة الهوية التراثية"
- لكل فرد صفحة كاملة تعرض: نسبه الكامل، إخوته، أبناءه، أحفاده، شارته، وموقعه في الشجرة

### د. إحصائيات جيلية
- "الجيل الثالث: 23 فرداً، 15 ذكراً و 8 إناث"
- رسم بياني يوضح توزيع الأجيال

---

## الركيزة 2: المشاركة والانتشار

### أ. Deep Linking (أولوية عالية)
- تحويل التطبيق لاستخدام React Router: `/#/person/:id` أو `/person/:id`
- عند فتح الرابط → يذهب مباشرة لبطاقة الشخص في عرض النسب
- هذا يجعل أي رابط يُشارك عبر الواتساب يفتح الشخص المطلوب مباشرة

### ب. بطاقة مشاركة WhatsApp (Killer Feature #1)
- زر "شارك نسبك" → يولّد صورة جميلة (Canvas API أو html2canvas) تحتوي:
  - اسم الشخص ونسبه الكامل
  - شعار العائلة
  - QR Code يحتوي رابط البوابة مع Deep Link للشخص
- تُحمّل كصورة PNG جاهزة للإرسال عبر WhatsApp

### ج. Open Graph Meta Tags
- عند مشاركة الرابط في WhatsApp، يظهر عنوان ووصف وصورة مناسبة
- عنوان ديناميكي: "نسب محمد بن زيد — بوابة آل الخنيني"

## الركيزة 3: تصور البيانات المتقدم

### أ. هيكلة بيانات الزوجات
- تحويل `spouses: string` إلى `spouses: { name: string; children_ids?: string[] }[]`
- هذا يسمح بعرض "أبناء لولوة العصيمي" و"أبناء مزنة البداح" منفصلين

### ب. مؤشرات بصرية للحالات الخاصة
- **متوفى صغيراً**: أيقونة نجمة صغيرة أو لون باهت على البطاقة
- **تعدد الزوجات**: عرض كل زوجة بخط ملون مختلف في الشجرة
- **فرع كبير vs صغير**: حجم العقدة يتناسب مع عدد الذرية

### ج. خريطة الأجيال (Timeline View)
- عرض أفقي زمني يرتب الأفراد على محور السنوات الهجرية
- يوضح التداخل الزمني بين الأجيال

## الركيزة 4: إمكانية الوصول لكبار السن

### أ. تعديلات فورية مطلوبة
- **حجم الخط**: زر تكبير/تصغير الخط (3 مستويات) يُحفظ في localStorage
- **أهداف اللمس**: التحقق أن كل زر لا يقل عن 44×44px (مطبق جزئياً حالياً)
- **تباين الألوان**: مراجعة نسب التباين وفق WCAG AA خصوصاً في الوضع الداكن

### ب. وضع "البساطة" (Simplified Mode)
- يُخفي كل الأقسام الثانوية ويعرض فقط: بحث → نتيجة → نسب
- خط أكبر، أزرار أكبر، معلومات أقل
- يُفعّل بزر واضح "الوضع المبسط" في صفحة الهبوط

### ج. إرشاد أول استخدام (Onboarding)
- عند أول زيارة: 3 شاشات بسيطة توضح كيف يبحث عن اسمه
- "اكتب اسمك هنا" → "اضغط على اسمك" → "هذا نسبك!"

### د. أمان التنقل
- زر "رجوع" واضح ودائم في كل الصفحات
- لا يوجد حالة "ضياع" — دائماً هناك طريق واضح للعودة

## الركيزة 5: بنية جاهزة للتوسع

### أ. تحضيرات فورية (بدون Supabase)
- **فصل البيانات عن العرض**: إنشاء طبقة `familyService.ts` تقرأ من `familyData.ts` الآن ومن API لاحقاً
- **React Query جاهز**: المكتبة مثبتة فعلاً — لف البيانات بـ `useQuery` حتى لو المصدر محلي
- **أنواع TypeScript محكمة**: إزالة `[key: string]: unknown` من `FamilyMember` وإضافة حقول مهيكلة

### ب. خطة Supabase (عند الجاهزية)
```text
Tables:
├── family_members (id, name, gender, father_id, birth_year, ...)
├── spouses (id, husband_id, wife_name, marriage_year)
├── spouse_children (spouse_id, child_id)  -- ربط الابن بأمه
├── user_roles (id, user_id, role)  -- admin/editor/viewer
└── audit_log (id, user_id, action, timestamp)

RLS Policies:
├── SELECT: public (الكل يقدر يشوف)
├── INSERT/UPDATE: admin + editor only
└── DELETE: admin only
```

### ج. لوحة إدارة (Admin Dashboard)
- إضافة/تعديل/حذف أفراد
- رفع صور شخصية
- سجل التعديلات (من عدّل ماذا ومتى)

## 3 ميزات قاتلة جديدة

### Killer Feature #1: "شارك نسبك" — بطاقة WhatsApp
صورة PNG فاخرة تُولّد تلقائياً تحتوي نسب الشخص الكامل بتصميم تراثي + QR Code. الأثر: كل فرد يشارك بطاقته → أفراد جدد يزورون البوابة → حلقة انتشار فيروسية.

### Killer Feature #2: "من أنا لك؟" — حاسبة القرابة
اختر شخصين → يظهر المسار بينهما بصرياً مع وصف العلاقة بالعربية الفصحى ("هو ابن عم أبيك"). الأثر: أكثر ميزة سيستخدمها الشباب في المجالس والتجمعات العائلية.

### Killer Feature #3: "شجرتي المصغرة" — عرض الفرع فقط
زر يعرض فقط فرع الشخص المحدد (أجداده + ذريته) كشجرة مستقلة نظيفة بدون بقية العائلة. مثالي لكبار السن الذين لا يريدون شجرة ضخمة بل فقط "عيالي وعيال عيالي". الأثر: يحل مشكلة التشتت في الشجرة الكبيرة ويعطي كل فرع إحساس بالخصوصية.
