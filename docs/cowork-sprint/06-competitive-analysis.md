```
الجانب الرأسي: سعر الاشتراك (من مجاني إلى مرتفع)
الجانب الأفقي: التركيز على الإدارة الحديثة ← → البحث التاريخي

                    [مرتفع الجودة + مرتفع السعر]

Ancestry ($25-50)                 MyHeritage ($129-299)
  [بحث تاريخي قوي]                [دولي + سهل]

                      FamilySearch (مجاني)
                    [بحث + تعاوني + مجاني]

                    [محلي + إدارة عائلية]

Ansab.io (غير معلن)            Nasb.io (غير معلن)
[ب٢ب محلي]                      [محلي، بسيط]

      Geni, WikiTree
      [مجاني + تعاوني]


                    *** نسبي (Nasaby) ***

                    [محلي + حديث + ذكي]
        RTL + Arabic-first + React Flow + Kinship Calc
        Multi-tenant + Admin Panel + Push Notifications
        $0-100 سنة (نموذج تسعير غير واضح بعد)
```

**الموضع الاستراتيجي للنسبي:**

1. **الفراغ:** لا يوجد منصة عربية متقدمة تجمع بين:
   - تفاعلية بصرية عالية (React Flow + d3)
   - حساب القرابة الذكي (بالعربية!)
   - إدارة شؤون العائلة (إشعارات، طلبات، تحديثات)
   - تطبيق جوال + Web معاً
   - محلية كاملة (RTL، نصوص عربية، أرقام عربية)
   - نموذج B2B (عائلات عديدة تحت منصة واحدة)

2. **الفرصة:** المنافسون المحليون (أنساب، نسب) حالياً إما غير واضحين أو بسطاء جداً. المنافسون العالميين مرتفعي السعر وغير محليين.

3. **التهديد الحقيقي:** ليس من المنافسين الموجودين، بل من:
   - تطبيق محلي جديد يأخذ نفس الفكرة (نسخ النموذج)
   - شركة ضخمة دولية تحسّن دعم اللغة العربية (Ancestry + MyHeritage)
   - حل مفتوح المصدر عربي قوي

## د) ١٠ أسباب يختار العميل نسبي — مرتبة من الأقوى

### ١. **التفاعلية البصرية الحقيقية**
   - شجرة عائلة تفاعلية بـ React Flow (تكبير، تصغير، سحب، تنقل حر)
   - رسم تلقائي ذكي مع d3-hierarchy (توليد المخطط الأمثل)
   - **لا يوجد منافس محلي يقدم هذا**
   - Ancestry / MyHeritage يقدمان نسخ ثابتة أقل تفاعل

### ٢. **حساب القرابة في الوقت الفعلي — بكلمات عربية صحيحة**
   - "أخوك من الأم" vs "ابن عمك" vs "جدك" — خوارزمية LCA مع 170+ حد قرابة عربي
   - لا أحد يفعل هذا بدقة الدرجة هذه محلياً أو عالمياً
   - **ميزة حصرية تقريباً**

### ٣. **محلية كاملة — لغة + تصميم + نموذج فكري**
   - واجهة RTL أساسية (كل شيء معكوس بشكل صحيح)
   - أرقام عربية ١٢٣ بدلاً من 123
   - تواريخ هجرية ١٤٤٧/٦/١٥ بدلاً من ميلادية
   - نصوص عربية مشكولة بـ Tatweel (تطويل ـ) لجمالية عربية حقة
   - **نموذج تفكير عائلي عربي** (أسماء بـ "بن/بنت"، صيغ النسب)

### ٤. **تطبيق جوال + Web كاملة**
   - PWA (Progressive Web App) — يعمل offline تماماً
   - تثبيت على الشاشة الرئيسية مثل التطبيق الأصلي
   - خادم Worker + Cache — بيانات محلية آمنة
   - Push Notifications — إشعارات حتى عند الإغلاق
   - **أفضل من التطبيقات الويب الأخرى الثقيلة**

### ٥. **شجرة واحدة لعائلة واحدة كاملة**
   - لا حاجة لـ "أشجار متعددة" (كما في Ancestry)
   - كل فرد عائلة يدخل شجرة واحدة موحدة
   - **أسهل وأنظف من التطبيقات الأخرى**

### ٦. **لوحة تحكم إدارية متطورة**
   - إضافة/حذف/تعديل أفراد بسهولة
   - مراجعة الطلبات (إضافة طفل، إضافة زوجة)
   - إرسال إشعارات مباشرة للأسرة
   - تتبع أنشطة الأسرة
   - **نموذج B2B محلي — صناديق الأسر والعائلات الكبيرة يحبونها**

### ٧. **خصوصية أولاً + تحكم كامل بالبيانات**
   - بيانات العائلة لا تغادر السيرفر السعودي
   - لا توجد شركة أجنبية تملك نسبك
   - رقابة مستخدم (من يرى الهاتف، من يرى السن)
   - **قيمة ثقافية وأمنية عميقة للسعوديين**

### ٨. **التسعير المحلي والعادل**
   - Ancestry: $25-50/شهر (بدون نفع للسعوديين لأنه لا سجلات عربية)
   - MyHeritage: $129-299/سنة (مرتفع)
   - **نسبي: نموذج تسعير عادل محلي** (ربما $0-100/سنة للعائلة)

### ٩. **النسب القديم يبقى محفوظاً**
   - Static Data في `familyData.ts` — النسب الأصلي آمن مئة بالمئة
   - البيانات Cloud تثري فقط، لا تمسح
   - **لا يمكن فقدان البيانات التاريخية**

### ١٠. **تصميم ومستقبل واضح**
   - فريق تقني سعودي فهم الثقافة والاحتياجات
   - خارطة طريق واضحة (توثيق أفضل، رسوم بيانية، فحص DNA محتمل)
   - **الاستثمار في منصة محلية بمستقبل بدل منصة أجنبية قديمة**

## هـ) كيفية منع التقليد — ٥ استراتيجيات واقعية

### ١. **حماية الملكية الفكرية + براءة الاختراع**

**الإجراء:**
- سجل براءة اختراع سعودية لـ:
  - خوارزمية حساب القرابة في الوقت الفعلي
  - تصميم الواجهة RTL مع d3-hierarchy
  - نموذج التسعير B2B للعائلات (multi-tenant)
- اطلب براءات دول الخليج (السعودية، الإمارات، الكويت)

**لماذا مهم:**
- أي منافس محلي يحاول نسخ سيتورط في مشكلة قانونية
- تثبيت السبق التقني

**التكلفة:** ~$5,000-10,000 (ضئيلة)

### ٢. **العلاقات المؤسسية المبكرة (B2B Moat)**

**الإجراء:**
- اتفاقيات طويلة الأجل مع صناديق الأسر الكبرى:
  - صندوق التميمي
  - صندوق الراجحي
  - صندوق الدويس
  - إلخ
- **عقود حصرية:** كل صندوق يستخدم نسبي + تسويق الكلمات الدعائية من داخل الصندوق

**لماذا مهم:**
- صناديق الأسر هي "اللاعبون الثقيلون" الذين يملكون المال والنفوذ
- منافس جديد سيجد الأسواق الرئيسية مغلقة بالفعل
- عقود الحصرية قد تكون شرط لتمويل أفضل

**التكلفة:** $ وقت في المبيعات، بدون تكلفة مباشرة

---

### ٣. **الابتكار المستمر — أسبق من المنافسين**

**الإجراء:**
- كل ٣ أشهر: ميزة جديدة حصرية:
  - Q2 2026: تحليل جينات الحمض النووي (DNA import)
  - Q3 2026: خريطة الهجرات التاريخية للعائلة
  - Q4 2026: صور بـ AI تلوين الصور العائلية القديمة
  - Q1 2027: فعاليات عائلية + تقويم مشترك
- **نشر المميزات الجديدة عبر وسائل التواصل بقوة**

**لماذا مهم:**
- "السبق التقني" يجعل المنافسين دائماً يلحقون، لا يقودون
- المستخدمون الحاليون يرون القيمة الجديدة كل ثلاث أشهر

**التكلفة:** ضمن الميزانية الهندسية الحالية

---

### ٤. **البيانات الضخمة — تأثير الشبكة (Network Effect)**

**الإجراء:**
- **كل عائلة تضيفها الآن = 100-500 شخص مدخل**
- بعد سنة واحدة: 50 عائلة × 250 شخص = 12,500 ملف شخصي مُدخل يدوياً
- هذه البيانات لا يستطيع منافس نسخها (ملك العائلات)
- الخوارزميات الذكية ستصبح أفضل (recommendations، matching)

**الإجراء العملي:**
- إنشاء "معرض العائلات" — شجرات عائلية مختارة تُعرض علناً (مع الموافقة)
- "أكبر شجرة عائلية عربية" — marketing claim قوي جداً

**لماذا مهم:**
- تأثير الشبكة: كل عائلة جديدة تجذب عائلات أخرى (المجتمع العائلي السعودي صغير)
- البيانات المتراكمة تخلق خندق قوي

**التكلفة:** وقت / رأس مال بشري

---

### ٥. **المجتمع والعلاقات الشخصية — "Stickiness"**

**الإجراء:**
- مجموعات Discord/WhatsApp عائلية (تدار من نسبي)
- **مسابقات:**
  - "أكبر شجرة عائلية" — جوائز
  - "أفضل قصة عائلية موثقة"
  - "أجمل صورة عائلية قديمة"
- ندوات حية (Webinars) مع متخصصي الأنساب من أرامكو، الجامعات
- **تطبيق لا يعتمد فقط على التكنولوجيا، بل على المجتمع**

**لماذا مهم:**
- المستخدمون يدفعون مقابل الميزات التقنية
- المستخدمون يعودون لأجل الماس والمجتمع
- صعوبة الانتقال = تحصين ضد الخروج

**التكلفة:** مدير مجتمع (راتب واحد)

## و) الخطوات الثلاث الأولى بكرة الصبح

### ١. اجتماع استراتيجي داخلي (90 دقيقة)

**من:** الفريق الإداري + الهندسي + المنتج

**الأجندة:**
- مراجعة سريعة لهذا التقرير
- **اتخاذ قرار:** هل نتقدم بطلب براءة اختراع سعودي؟
- **اتخاذ قرار:** من يمتلك العلاقة مع أول صندوق أسرة؟ (واجبة توكيل)
- **تحديد:** كم عائلة نستهدف بنهاية Q2 2026؟ (رقم صغير = 5-10 عائلات)
- **تحديد:** أول 3 ميزات جديدة للـ Q3؟ (DNA، الهجرات، التلوين بـ AI)

### ٢. اتصال مباشر مع أول عميل — عائلة الخنيني (ساعة واحدة)

**الموضوع:**
- توضيح أن نسبي **محلي 100%** وآمن
- عرض Live demo لـ Al-Khunaini (410+ أفراد بالفعل في النظام!)
- **اقتراح:** هل تريدون أن تكونوا "عائلة رائدة" (الأولى على نسبي)؟
- أثناء العرض: اطلب ملاحظات، واطلب إذن لاستخدام عائلتك كـ "case study" في التسويق

**الهدف:**
- عائلة الخنيني = testimonial قوي جداً (410+ فرد يعني المصداقية)
- صور + فيديو = مادة تسويقية

---

### ٣. إعداد قائمة المبيعات للخطوة التالية (ساعة واحدة)

**المهام:**
- **اجمع 20 صندوق أسري سعودي كبير** (التميمي، الراجحي، الدويس، الخنيني، الأحمد، إلخ)
- ابحث عن مدير العلاقات العام لكل صندوق
- **أرسل رسالة تقديم مخصصة** (لا رسالة نسخ):
  - "لقد أنشأنا نسبي خصيصاً للعائلات السعودية الممتدة"
  - "فيها حساب القرابة بالعربية، واجهة RTL كاملة، وخصوصية مطلقة"
  - "هل يمكن لقاء قصير الأسبوع القادم لعرض مثال حي؟"
- **الهدف:** 3 اجتماعات في الأسبوع المقبل
