## أ) هل يحتاج سجل تجاري الحين؟

### الجواب المختصر
**لا — ليس فوراً.** لكن في **نقاط محددة** يصبح إجباري. تفصيل:

### متى يصبح السجل التجاري إجباري؟

#### ١. استقبال الدفع الأول (الآن)
- **الحالة الحالية**: أنت تختبر الفكرة، صفر عائلات مدفوعة
- **القانون السعودي**:
  - أي شخص **يمارس نشاط تجاري بقصد الربح** يجب أن يسجل سجل تجاري
  - "نشاط تجاري" = تقديم خدمة بمقابل مالي بشكل منتظم
  - حتى **دفعة واحدة** من عائلة تُحسب "نشاط تجاري"
- **النتيجة**: من اللحظة التي تستقبل فيها أول دفعة، تصبح بحاجة لسجل تجاري **قانونياً**

#### ٢. فتح حساب بنكي للعائلة (شهر واحد من الآن)
- **البنوك السعودية** لن تفتح حساب تجاري بدون سجل تجاري
- إذا حاولت تحويل أموال من **حسابك الشخصي** → تحتمل الإيقاف/التحقيق من البنك
- **الحل**: احصل على السجل قبل فتح حساب تجاري

#### ٣. التوسع (شهر ٣-٤)
- إذا وصلت لـ ٥+ عوائل مدفوعة، التفتيش التجاري قد يبدأ يسأل أسئلة
- إذا كان لديك موظفين (حتى متعاقدين)، يصبح إجباري 100%

### الخطوات والتكاليف (السعودية ٢٠٢٦)

#### الطريقة الرسمية: السجل التجاري من وزارة التجارة

**المراحل**:

1. **الخطوة ١: التسجيل عبر منصة العامة (my.gov.sa)** — ٣٠ دقيقة
   - اذهب: https://my.gov.sa/en/services/240897
   - اختر "السجل التجاري للمنشأة"
   - أدخل اسم النشاط: "تطبيق ويب لشجرة العائلة" أو "خدمات رقمية"
   - رفع هويتك الوطنية صورة واضحة
   - اختر عنوان المقر (يمكن عنوانك الشخصي)
   - التكلفة الأولية: **لا شيء** (فقط الرسوم الحكومية)

2. **الخطوة ٢: الموافقة والسداد** — يوم واحد
   - وزارة التجارة تراجع الطلب (١-٣ أيام عملية)
   - تتلقى إشعار بالموافقة
   - تسدد الرسوم الحكومية: **~٥٠٠-٨٠٠ ريال** (تقريباً)

3. **الخطوة ٣: استلام السجل** — ٣-٥ أيام عملية
   - تحصل على رقم السجل التجاري (يبدأ بـ "١" وسته أرقام)
   - تطبع نسخة من البوابة
   - جاهز للاستخدام

**الوقت الكلي**: ١-٢ أسبوع من البداية للسجل بيدك

**التكاليف الكلية**:
- رسوم حكومية: ٥٠٠-٨٠٠ ريال
- استشارة محامي (اختياري): ٠ ريال (تستطيع بدون)
- **المجموع**: ٥٠٠-٨٠٠ ريال فقط

#### مصادر رسمية:
- [وزارة التجارة — خدمة السجل التجاري](https://my.gov.sa/en/services/240897)
- [منصة الأعمال السعودية — دليل التسجيل](https://business.sa/en/servicesprocedures/details/3f37df76-a853-4616-ef34-08dced12ee80)

### البدائل المؤقتة (قبل السجل التجاري)

إذا كنت تريد **تجنب السجل للآن** والانتظار:

#### ١. التحويل البنكي الشخصي (الخطر)
- اطلب من العائلة تحويل المال لحسابك الشخصي
- **المشكلة**:
  - البنك قد يسأل "من أين هذا المال؟"
  - إذا اكتشفوا نشاط تجاري، قد يوقفوا الحساب
  - **غير قانوني** إذا كانت التحويلات منتظمة
- **المدة الآمنة**: دفعة أو اثنتين فقط
- **التصنيف**: ⚠️ لا أنصح

#### ٢. PayPal (خطأ — لا يعمل في السعودية)
- PayPal **لا تقبل مستقبلين في السعودية** منذ ٢٠١٨
- استبدلت بـ "2Checkout" لكنها معقدة
- **التصنيف**: ✗ غير عملي

#### ٣. Mada/بطاقة الصراف (غير مناسب)
- عملية يدوية، بطيئة، غير آمنة
- **التصنيف**: ✗ للعائلات ٢-٣ فقط

#### ٤. **الحل الأفضل المؤقت**: ابدأ مع Tap الآن (بدون سجل)
- Tap تقبل الدفع **قبل** السجل التجاري (اختبار)
- فور وصول عائلة واحدة جدية → احصل على السجل بسرعة
- Tap ستطلب السجل **عند الحد من الرسوم أو التوسع**
- **الوقت**: أسبوعين اختبار قبل السجل آمن
- **التصنيف**: ✅ الأفضل

### الرسوم (التفاصيل)

#### رسم الدفعة الواحدة
```
المبلغ: ١,٠٠٠ ريال (مثال)
الرسم: ٢.٨٥% = ٢٨.٥٠ ريال
الرسم الثابت: ٠.٣٠ ريال
الإجمالي المخصوم: ٢٨.٨٠ ريال
المبلغ لحسابك: ٩٧١.٢٠ ريال

نسبة الخسارة: ٢.٨٨%
```

#### بدون حد أدنى للمبلغ
- دفعة بـ ١٠ ريال؟ تدفع ٠.٣ ريال فقط (لا رسم نسبي)
- دفعة بـ ١,٠٠٠,٠٠٠ ريال؟ نفس ٢.٨٥%

#### مقارنة مع المنافسين

| البوابة | الرسم | على ١,٠٠٠ ر | على ١٠,٠٠٠ ر |
|--------|-------|-----------|-----------|
| **Tap** | ٢.٨٥% + ٠.٣٠ | ٢٨.٨٠ | ٢٨٥.٣٠ |
| **دفاع** | ٢.٩٩% ثابت | ٢٩.٩٠ | ٢٩٩.٠٠ |
| **Payfort** | ٢.٩٩% + متغير | ~٣٠ | ~٣٠٠ |

**الفرق**: Tap توفر ١-٢ ريال لكل دفعة صغيرة. مع ٣٠ عائلة = توفير ٣٠-٦٠ ريال/شهر.

##### ٢. إنشاء مكون الدفع (React)

ملف جديد: `src/components/PaymentModal.tsx`

```typescript
import React, { useRef, useEffect } from 'react';

// استورد Tap SDK
declare global {
  interface Window {
    Tap?: any;
  }
}

interface PaymentModalProps {
  amount: number;           // المبلغ بالريالات
  familyId: string;         // معرّف العائلة
  onSuccess: (transactionId: string) => void;
  onError: (error: string) => void;
}

export function PaymentModal({
  amount,
  familyId,
  onSuccess,
  onError,
}: PaymentModalProps) {
  const containerRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    // ١. تحميل سكريبت Tap
    const script = document.createElement('script');
    script.src = 'https://sdk.tap.company/v2/tap.js';
    script.onload = () => {
      initializeTapPayment();
    };
    document.body.appendChild(script);

    return () => {
      document.body.removeChild(script);
    };
  }, []);

  const initializeTapPayment = () => {
    if (!window.Tap) return;

    const publicKey = import.meta.env.VITE_TAP_PUBLIC_KEY;

    // ٢. إنشاء خيارات الدفع
    const elements = window.Tap.Elements({
      publicKey,
    });

    const cardElement = elements.create('card', {
      labels: {
        cardNumber: 'Card Number',
        cardExpiry: 'MM/YY',
        cardCvc: 'CVC',
      },
      placeholder: {
        cardNumber: '4111 1111 1111 1111',
        cardExpiry: 'MM/YY',
        cardCvc: '123',
      },
    });

    // ٣. تركيب البطاقة
    if (containerRef.current) {
      cardElement.mount(containerRef.current);
    }

    // ٤. معالج الدفع
    const handlePayment = async () => {
      try {
        const { token } = await cardElement.createToken();

        // ٥. إرسال التوكن للسرفر
        const response = await fetch('/api/process-payment', {
          method: 'POST',
          headers: { 'Content-Type': 'application/json' },
          body: JSON.stringify({
            token,
            amount,
            familyId,
            currency: 'SAR',
          }),
        });

        const data = await response.json();

        if (data.success) {
          onSuccess(data.transactionId);
        } else {
          onError(data.error);
        }
      } catch (error) {
        onError(error instanceof Error ? error.message : 'Payment failed');
      }
    };

    // ربط الزر بمعالج الدفع
    const payButton = document.getElementById('pay-button');
    if (payButton) {
      payButton.addEventListener('click', handlePayment);
    }
  };

  return (
    <div className="space-y-4">
      <div ref={containerRef} className="border rounded-lg p-4 bg-white" />
      <button
        id="pay-button"
        className="w-full bg-blue-600 text-white py-2 rounded-lg font-semibold"
      >
        ادفع {amount} ريال
      </button>
    </div>
  );
}
```

##### ٣. إعداد Edge Function للمعالجة (Supabase)

ملف جديد: `supabase/functions/process-payment/index.ts`

```typescript
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
)

serve(async (req) => {
  if (req.method !== 'POST') {
    return new Response('Method not allowed', { status: 405 })
  }

  try {
    const { token, amount, familyId, currency } = await req.json()

    // ١. استدعاء Tap API
    const tapResponse = await fetch('https://api.tap.company/v2/charges', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${Deno.env.get('TAP_SECRET_KEY')}`,
      },
      body: JSON.stringify({
        amount: Math.round(amount * 100), // بالفلوس
        currency,
        source: {
          id: token,
        },
        reference: {
          transaction: `nasaby_${familyId}_${Date.now()}`,
        },
        receipt: {
          email: true, // أرسل إيصال للعميل
        },
      }),
    })

    const tapData = await tapResponse.json()

    if (tapData.id && tapData.status === 'CAPTURED') {
      // ٢. حفظ في Supabase
      await supabase.from('payments').insert({
        family_id: familyId,
        tap_transaction_id: tapData.id,
        amount,
        status: 'completed',
        created_at: new Date(),
      })

      return new Response(JSON.stringify({
        success: true,
        transactionId: tapData.id,
      }), { status: 200 })
    } else {
      return new Response(JSON.stringify({
        success: false,
        error: tapData.message || 'Payment failed',
      }), { status: 400 })
    }
  } catch (error) {
    return new Response(JSON.stringify({
      success: false,
      error: error.message,
    }), { status: 500 })
  }
})
```

##### ٤. جدول Supabase للدفعات

```sql
-- في Supabase SQL Editor
create table payments (
  id uuid primary key default gen_random_uuid(),
  family_id text not null references family_members(id),
  tap_transaction_id text unique not null,
  amount integer not null, -- بالفلوس (x100)
  status text not null check (status in ('pending', 'completed', 'failed')),
  created_at timestamp with time zone default now(),
  updated_at timestamp with time zone default now()
);

-- فهرس للبحث السريع
create index idx_payments_family on payments(family_id);
create index idx_payments_status on payments(status);
```

المرحلة ٢: إنشاء Subdomain (يوم +١ إلى +٢)
───────────────────────────────────────────────────────────────────

[ ] ٤. جمع اسم العائلة (في البريد الأول أو جديد)
    ├─ اطلب: "اسم العائلة الذي سيظهر في الرابط؟"
    ├─ مثال: "Al-Khunaini" → family-khunaini.nasaby.app
    ├─ التحقق: فريد ولم يستخدم
    └─ الوقت: ٣٠ دقيقة انتظار الرد

[ ] ٥. إنشاء Subdomain (فني)
    ├─ إضافة DNS record في Vercel:
    │  CNAME: family-khunaini.nasaby.app → nasaby.app
    ├─ الانتظار لـ DNS propagation: ١٥-٣٠ دقيقة
    ├─ اختبار: curl family-khunaini.nasaby.app
    └─ الوقت: ٢٠ دقيقة

[ ] ٦. إنشاء حساب admin للعائلة
    ├─ في لوحة التحكم: إضافة admin جديد
    ├─ البيانات: name, email, phone
    ├─ كلمة السر المؤقتة: random + احفظها
    ├─ تفعيل الصلاحيات: view_all_members, edit_members
    └─ الوقت: ١٠ دقائق

[ ] ٧. إرسال بيانات الوصول
    ├─ بريد للعائلة يتضمن:
    │  ├─ الرابط: family-khunaini.nasaby.app
    │  ├─ اسم المستخدم: [المعطى]
    │  ├─ كلمة السر المؤقتة: [المعطاة]
    │  └─ تعليمات: "غيّر كلمة السر أول مرة"
    └─ الوقت: ٣ دقائق

───────────────────────────────────────────────────────────────────
المرحلة ٣: إدخال البيانات (يوم +٣ إلى +١٠)
───────────────────────────────────────────────────────────────────

[ ] ٨. تسليم نموذج جمع البيانات
    ├─ شيت Excel/Google Sheets يتضمن:
    │  ├─ أعضاء العائلة (الاسم، التاريخ الميلادي)
    │  ├─ العلاقات (الأب، الأم، الزوج/ة)
    │  ├─ معلومات الاتصال (اختياري)
    │  └─ صور (اختياري)
    ├─ تعليمات واضحة للملء
    └─ الوقت: ١ دقيقة إرسال

[ ] ٩. استقبال البيانات من العائلة
    ├─ انتظر حتى ٥ أيام
    ├─ تذكير بريد يوم ٤ إذا لم يرسلوا
    └─ الوقت: تنتظر

[ ] ١٠. التحقق من صحة البيانات
    ├─ هل جميع الأسماء موجودة؟
    ├─ هل التواريخ بصيغة صحيحة؟
    ├─ هل العلاقات منطقية؟ (الأب أكبر من الأطفال)
    ├─ طلب توضيحات إن لزم
    └─ الوقت: ٣٠ دقيقة

[ ] ١١. إدخال البيانات في النظام
    ├─ تحويل البيانات من الشيت إلى CSV
    ├─ رفع عبر لوحة التحكم أو API
    ├─ التحقق المرة الأخيرة من ظهور الأعضاء
    └─ الوقت: ٣٠ دقيقة — ١ ساعة

───────────────────────────────────────────────────────────────────
المرحلة ٤: الاختبار والتفعيل (يوم +١١)
───────────────────────────────────────────────────────────────────

[ ] ١٢. اختبار الشجرة
    ├─ فتح family-khunaini.nasaby.app
    ├─ تحقق من ظهور جميع الأعضاء
    ├─ اختبر البحث بالأسماء
    ├─ اختبر حساب القرابة بين أعضاء
    ├─ اختبر عرض الأجيال
    └─ الوقت: ٢٠ دقيقة

[ ] ١٣. تفعيل للعائلة
    ├─ بريد النهاية السعيدة:
    │  "شجرتك الخاصة جاهزة ✓"
    ├─ رابط الوصول المباشر
    ├─ طلب أي تصحيحات
    └─ الوقت: ٣ دقائق

───────────────────────────────────────────────────────────────────
المرحلة ٥: المتابعة (أسابيع ١-٤)
───────────────────────────────────────────────────────────────────

[ ] ١٤. بريد المتابعة (يوم +١٤)
    ├─ "كيف تجربتك بعد أسبوعين؟"
    ├─ هل تحتاج إضافة أعضاء؟
    ├─ هل هناك مشاكل؟
    └─ الوقت: ٢ دقيقة

[ ] ١٥. بريد الشهر (يوم +٣٠)
    ├─ "شكراً على دعمك — نسبي"
    ├─ نصائح استخدام جديدة
    ├─ طلب رأيهم (feedback)
    └─ الوقت: ٣ دقائق

[ ] ١٦. بريد التحديث (يوم +٩٠)
    ├─ إذا كانوا نشطين:
    │  "فيّ ميزات جديدة في الطريق"
    ├─ إذا لم يستخدموا كثيراً:
    │  "كيف نحسّن الخدمة لك؟"
    └─ الوقت: ٣ دقائق

### جدول الأوقات المنتظرة

| الخطوة | الوقت المقدّر | المرونة | ملاحظات |
|--------|------------|--------|--------|
| **الدفع والتأكيد** | فوري - ١ ساعة | ± ٣٠ دقيقة | يعتمد على بنك العائلة |
| **DNS Propagation** | ١٥-٣٠ دقيقة | حتى ساعتين | أحياناً فوري |
| **انتظار البيانات** | ٣-٥ أيام | حتى أسبوع | غالب العوائل تأخر |
| **إدخال البيانات** | ٣٠ دقيقة - ١ ساعة | حسب العدد | ٥٠ عضو = ١ ساعة |
| **الاختبار** | ٢٠ دقيقة | ± ١٠ | قد تحتاج تصحيحات |

### الوقت الإجمالي من البداية للنهاية

```
يوم ١:     ١-٢ ساعة (دفع + تأكيد + بريد)
يوم ٢-٣:   ١ ساعة (subdomain + admin + بيانات وصول)
يوم ٤-١٠: انتظار + ٣٠ دقيقة (جمع بيانات + تحقق)
يوم ١١:    ١.٥ ساعة (إدخال + اختبار + تفعيل)

───────────
المجموع:  ~١٢ يوم تقويمي
العمل الفعلي: ~٤ ساعات
المستخدم ننتظره: ٨ أيام
```

### مثال زمني واقعي

```
الإثنين ١ أبريل، الساعة ٨ صباحاً:
  ✉️ بريد من عائلة الخنيني
  ├─ ٩:٠٠ صباحاً: ترد بالتأكيد + رابط الدفع
  └─ ١٠:٣٠ صباحاً: استقبال الدفع من Tap ✓

الثلاثاء ٢ أبريل، الساعة ١٠ صباحاً:
  ├─ ١٠:٣٠ صباحاً: بريد بـ subdomain
  ├─ ١١:٠٠ صباحاً: DNS ready ✓
  └─ ١٢:٠٠ ظهراً: بيانات الوصول للعائلة

الأربعاء ٣ أبريل إلى الجمعة ١٠ أبريل:
  └─ انتظار + تذكير

السبت ١١ أبريل، الساعة ٣ عصراً:
  ├─ استقبال بيانات العائلة ✓
  ├─ ٣:٣٠ عصراً: إدخال البيانات
  ├─ ٤:٣٠ عصراً: الاختبار ✓
  └─ ٥:٠٠ عصراً: بريد التفعيل

النتيجة: ١٠ أيام من البريد الأول للتفعيل الكامل
```
