## المقدمة

تحتاج منصة نسبي إلى استراتيجية نمو قوية للتطور من "عائلة واحدة واعدة" إلى "حركة عائلية سعودية". هذا المستند يحدد آليات النمو الفيروسي والحلقات اللاصقة والإجراءات التي تجعل المستخدم يعود كل يوم، كل أسبوع، كل شهر.

## 1. النمو الفيروسي (Viral Growth Loops)

### المشكلة الأساسية
عائلة واحدة (الخنيني) بـ ٤١٠+ عضو = حدود محدودة جداً. الهدف: تحويل كل عضو راضٍ إلى **سفير للمنصة** يجلب عائلته.

### آلية 1.1: "دعوة العائلة الأخرى"

**الحالة الحالية:**
- التطبيق مغلق أمام العائلات الأخرى
- رابط الدعوة يقول: "أطلب من عائلتك أن تنضم"

**الحلقة الفيروسية:**
```
عضو خنيني راضٍ
    ↓
عضو يشوف شجرة عائلته الجميلة
    ↓
"يا جماعة، هذا التطبيق رهيب!" (مشاركة في الواتس)
    ↓
أقاربه يضغطون تحميل التطبيق
    ↓
يسجلون عائلتهم (عائلة جديدة أولى!)
    ↓
كل واحد فيهم يدعو شخصين آخرين
    ↓
نمو أسي
```

**التنفيذ:**
1. **زر "شارك العائلة"** بارز جداً على الصفحة الرئيسية:
   ```
   🔗 شارك شجرة عائلتك مع الأقارب الآخرين
   ┌─────────────────────────────┐
   │ انسخ الرابط | شارك على الواتس │
   └─────────────────────────────┘
   ```

2. **الرابط الديناميكي:**
   - ينقل إلى صفحة استقبال: `nasaby.app/join-family?code=KHUNAINI_001`
   - تعرض: اسم العائلة، عدد الأعضاء، صورة الشجرة (preview)

3. **شهادات حقيقية:**
   - عرض تعليقات من أعضاء: "والله هاذا التطبيق غيّر إحساسي بالعائلة"
   - صور من الاحتفالات العائلية المتصلة بالمنصة

### آلية 1.2: "تحدي النسب" (Genealogy Challenge)

**الفكرة:**
- اللاعب يحسب قرابة شخصين عشوائيين
- إذا أجاب صح، يشارك النتيجة على الواتس: "أنا أحسب النسب! 🧬 هل تقدر أنت؟"
- الصديق يضغط الرابط → يحتاج ينزل التطبيق ليلعب

**التنفيذ:**
```javascript
const challengeLink = `
nasaby.app/challenge?from=محمد&to=علي&familyCode=KHUNAINI
شارك على: WhatsApp | Telegram | X | Instagram
```

## 2. حلقة النمو الكاملة (Growth Flywheel)

```
عضو جديد
  │
  ├─ تسجيل عبر الباركود أو الرابط
  │
  ├─ يشوف عائلته كاملة في ٣٠ ثانية ✨
  │
  ├─ الشعور: "يالا كل العائلة هنا!"
  │
  ├─ يسجل رقم الجوال + ميلاده
  │
  ├─ يستقبل إشعار: "مرحباً بك، لقد عثرنا على X أقارب لك"
  │
  ├─ يتصفح الشجرة، يحسب قرابة، يضيف مولود جديد
  │
  ├─ الإجراء الحاسم: يضغط "شارك مع قريب" ← 🔑 **فيروسي**
  │
  ├─ يُنسخ اسمه على الواتس كـ "محمد دعاك لتصفح عائلتك"
  │
  └─ القريب ينزل التطبيق → العضو الجديد التالي
```

## 3. Gamification: نظام النقاط والشارات

### المبدأ: "تحويل التراث إلى لعبة ممتعة"

يجب أن تكون جميع الشارات والنقاط والإنجازات **محترمة ثقافياً وعائلياً**، وليست مثل ألعاب الفيديو السخيفة.

### 3.1 نظام النقاط

| الإجراء | النقاط | التفسير |
|--------|-------|--------|
| تسجيل الدخول أول مرة | ٥ | ترحيب |
| إضافة صورة شخصية | ١٠ | تطوير الملف الشخصي |
| تسجيل تاريخ ميلاد هجري | ١٥ | بيان الهوية الثقافية |
| إضافة مولود جديد | ٢٠ | إثراء الشجرة |
| إضافة زوجة/زوج | ١٥ | تسجيل الصلات |
| حساب قرابة مع شخص | ٥ | التفاعل |
| شراء نسخة عائلية (العائلة) | ١٠٠ | دعم الإيكوسيستم |
| دعوة ٥ أعضاء جدد | ٥٠ | نمو فيروسي ⭐ |
| دعوة عائلة جديدة كاملة | ٢٠٠ | نمو بطيء لكن مستدام |

**نقاط التحويل:**
- ١٠٠ نقطة → حصول على خصم ١٠% على خطة العائلة
- ٥٠٠ نقطة → شهادة "راعي النسب" (PDF جميلة بطابع عربي)

### 3.2 الشارات (Badges)

| الشارة | الشرط | الوصف |
|-------|-------|--------|
| 🌳 **مؤسس الشجرة** | أول ١٠ أعضاء سجّلوا أنفسهم | "أنت من أسس هذه الشجرة العائلية" |
| 🔍 **مؤرخ العائلة** | أضاف ٢٠+ معلومة صحيحة | يظهر اسمه كـ "موثّق" |
| 👶 **منجب مبارك** | أضاف ١٠+ أطفال للشجرة | "مباركة على صلتك بالعائلة" |
| ♥️ **حافظ العهد** | سجل بيانات ٥+ أقارب براضاهم | ثقة المجتمع |
| 🎓 **عالم النسب** | حسب قرابة ٥٠+ شخص | "إنك خبير!" |
| 🌍 **رسول العائلة** | دعا عائلة جديدة كاملة | "ساهمت في توسيع مجتمع نسبي" |
| 📸 **مصور التراث** | أضاف صور عائلية ١٠+ | شهادة تقدير |
| 🏛️ **الحارس الأمين** | استخدم التطبيق ٩٠ يوم متتالية | "أنت ملتزم بالعائلة" |

**عرض الشارات:**
- تظهر على ملف المستخدم: "محمد — 🌳 مؤسس الشجرة 🎓 عالم النسب"
- تظهر في قوائم البطاريق الأسبوعية

### 3.3 لوحات الصدارة (Leaderboards)

```
🏆 أنشط الأعضاء هذا الأسبوع

1️⃣  محمد الخنيني     | ٣٧٥ نقطة | 🌳 🔍 👶
2️⃣  نورة بنت علي    | ٣١٠ نقطة | 🔍 🎓
3️⃣  فهد الدويش      | ٢٨٥ نقطة | 🌍 📸

🏅 هل تريد أن تكون في القمة؟ أضف مزيد من البيانات!
```

**الفئات:**
- النشاط العام
- الدعوات الفعالة (من أضاف أكتر أعضاء؟)
- أعمار الأعضاء (تحدي بين الأجيال: "من الأكتر نشاطاً: الشباب أم الكبار؟")

## 4. أسباب العودة: "لماذا أفتح التطبيق كل يوم؟"

### المشكلة الأساسية:
بعد التسجيل الأول والفضول الأولي، المستخدم نسي الرابط. ماذا يجعله يعود؟

### 4.1 الحافز اليومي

**الزائرون اليوميون (Day-1, Day-7, Day-30):**

| اليوم | الحافز | الإجراء |
|-------|--------|--------|
| اليوم ١ | فضول مشروع | شوف شجرتك + ادعُ شخص واحد |
| اليوم ٣ | إشعار: "قريبك محمد سجّل، أضاف ابن جديد" | اضغط، شوف التحديث |
| اليوم ٧ | تحدي أسبوعي: "احسب قرابة قريبك" | لعبة سريعة |
| اليوم ١٤ | إشعار عيد ميلاد قريب: "تهنئة ياسر بعيد ميلاده" | شعور شخصي |
| اليوم ٣٠ | "استعرض شهرك: أضفت X مولود، دعوت Y شخص" | يذكرك بإنجازاتك |

### 4.2 لحظات العودة الموسمية

| الحدث | السلوك | الإشعار |
|-------|--------|--------|
| **عيد الأضحى / الفطر** | عائلات تلتقي | "تم تسجيل ١٢ عضو جديد من عائلتك! 🎉" |
| **زواج** | عائلة متسعة | "مبارك الزواج! تم تسجيل X عضو جديد" |
| **ولادة** | إضافة وشم | تذكير: "سجل مولودك الجديد في الشجرة!" |
| **اجتماع عائلي** | نشاط مرتفع | "أكتر من ٥٠ عضو نشط اليوم 🔥" |
| **ذكرى وفاة** | استرجاع الذكريات | "سجّل ذكريات مع والدك (١٩٥٠–٢٠١٥)" |

## 5. الإشعارات الذكية (Smart Notifications)

### الوضع الحالي:
- الإشعارات موجودة: approval, rejection, broadcast, new_member, info
- لكن **نادراً ما تُستخدم لتحفيز العودة**

### 5.1 أنواع الإشعارات الجديدة

#### أ) إشعارات "التذكير بالقرب"
```
عندما يدخل قريبك للتطبيق:
👤 محمد الخنيني دخل الآن
📍 نسبي
👉 اضغط لتشوف نشاطه الأخير
```

#### ب) إشعارات الذكريات
```
📅 ٣ ربيع الأول = ميلاد والدتك أم محمد
❤️ مارس ١٩٨٩ — صحتها دايمة
👉 أضف صورة أو ذكرى من ذاك الحين
```

#### ج) إشعارات التنبيه العائلي
```
⚠️ تم اكتشاف معلومة ناقصة:
🔴 والدة (أم) الخنيني غير معروفة بعد

هل عندك المعلومة؟ ساعدنا ننكمل الشجرة!
```

#### د) إشعارات المنجزات
```
🏆 مبارك!
أنت الآن راعي النسب:
✓ أضفت ٥٠+ عضو
✓ دعوت ٣ عائلات
✓ حسبت ١٠٠ قرابة

شارك إنجازك على الواتس 📲
```

#### هـ) الإشعارات الاجتماعية (Social FOMO)
```
🔥 اشتعل!
١٢ عضو دخلوا الآن
شجرة الخنيني اليوم الأكتر نشاط!

استمع معهم 👊
```

### 5.2 جدول الإشعارات المثلى

```
الساعة 09:00 صباحاً  → "صباح الخير! قريبك X سجّل اليوم"
الساعة 01:00 ظهراً  → "تحدي الظهيرة: احسب قرابة ثالثة لتكسب نقاط"
الساعة 06:00 مساءً  → "ياسر دخل التطبيق الآن 👋 شنو أخباره؟"
الساعة 09:00 مساءً  → "ملخص يومك: ٢ نشاط جديد من الخنيني"
الساعة الجمعة 08:00 صباحاً → "استعداد للاجتماع العائلي؟ مين حيحضر من التطبيق؟"
```

**القاعدة الذهبية:**
- لا أكتر من ٣ إشعارات في اليوم (إلا في الأحداث الكبرى)
- كل إشعار **يجب أن يكون شخصياً** (ليس broadcast عام)
- كل إشعار **يجب أن يجر لإجراء** (لا تشعارات فارغة)

## 6. الفعاليات العائلية الرقمية (Digital Family Events)

### الفكرة الأساسية:
التطبيق لا يجب أن يكون مجرد **أرشيف**. يجب أن يكون **ساحة اجتماع عائلي رقمية**.

### 6.1 "تحدي النسب الأسبوعي" (Weekly Kinship Challenge)

**الآلية:**
```
كل جمعة الساعة ٨ صباحاً:
📢 تحدي الأسبوع: احسب قرابة شخصين عشوائيين

مثال:
❓ ما قرابة محمد الخنيني من نورة بنت زيد؟

خيارات:
A) أخ  B) عم  C) ابن عم  D) ابن خال

⏱️ عندك دقيقة واحدة!

👑 أول ٣ أجوبة صحيحة يحصلون على نقاط إضافية
```

### 6.2 "ليلة العائلة الرقمية" (Family Night)

**متى؟** كل جمعة الساعة ٧ مساءً (موعد جماعي)

**الحدث:**
1. جميع أعضاء الخنيني يدخلون التطبيق
2. بث مباشر صغير: "من يتحدث عن أقدم ذكرى في الشجرة؟"
3. الأعضاء يرسلون أسئلة لبعضهم البعض (Q&A)
4. أفضل قصة تُكسب ١٠٠ نقطة
5. صورة عائلية جماعية من الحضور

**المنصة الحالية:** احفظ هذا في التطبيق (لاحقاً: Zoom integration)

### 6.3 "حملة الوثائق العائلية" (Document Campaign)

**كل شهر:**
```
📸 "شهر الصور التراثية"
شارك أقدم صورة عندك (والدك، والديك، جدك)
👑 أجمل صورة تظهر في معرض نسبي

📝 "شهر القصص"
اكتب قصة عن شخص في الشجرة
🏆 أفضل قصة تُنشر في النشرة الإخبارية الشهرية
```

### 6.4 "مسابقة الشجرة الأكثر نشاطاً" (Branch Competition)

```
🏆 فرع محمد vs فرع ناصر vs فرع عبدالعزيز

📊 المقاييس:
- من يسجل أكتر أعضاء هذا الشهر؟
- من يضيف أكتر بيانات صحيحة؟
- من يدعو أكتر عائلات جديدة؟

🎁 الفرع الرابح:
- شهادة تقدير رسمية
- دعوة خاصة لاجتماع عائلي رسمي
```

## 7. معدلات الاحتفاظ المستهدفة (Retention Targets)

### المعادلة الأساسية:
```
Day-1 (D1):   ≥ 40%  ← من سجّل يفتح في اليوم الأول
Day-7 (D7):   ≥ 25%  ← ربع الأعضاء الجدد يعودون في الأسبوع
Day-30 (D30):  ≥ 15%  ← ١ من كل ٧ أعضاء جدد يستخدم شهرياً
Day-90 (D90):  ≥ 10%  ← نواة المستخدمين الدائمين
```

### تقسيم المستخدمين حسب السلوك:

| النوع | الوصف | النسبة المستهدفة | الاستراتيجية |
|------|--------|-------------------|----------------|
| **Core Users** | يستخدمون كل يوم أو يوم بعد يوم | ٥–١٠% | نمو فيروسي + challenges |
| **Active Users** | يستخدمون أسبوعياً | ١٥–٢٥% | إشعارات ذكية + فعاليات |
| **Casual Users** | يستخدمون شهرياً | ٢٠–٣٠% | تذكيرات موسمية |
| **Dormant Users** | لم يفتحوا في ٣٠ يوم | ٣٠–٤٠% | حملة إعادة تفعيل |

### الهدف النهائي:
في سنة واحدة من تسجيل عائلة جديدة:
- ١٥% من أعضاء العائلة الجديدة يصبحون "core users"
- ٣٠% يصبحون "active users"
- بقية الـ ٥٥% يظلون "casual" أو "dormant" (مقبول — تكلفة الحفاظ منخفضة)

## 8. استراتيجية "العائلة النائمة" (Sleeping Family Recovery)

### المشكلة:
عائلة سجلت ٢٠ عضو، أضافوا بيانات، لكن بعد ٢ شهر اختفوا تماماً.

### 8.1 رحلة إحياء العائلة (3-Week Reactivation Campaign)

#### الأسبوع الأول:
```
البريد الإلكتروني (أو push notification):
"أين ذهبت يا عائلة الخنيني؟ 👋

آخر نشاط: ١ سبتمبر
أضفتم معاً: ٢٣ عضو جديد ✓

اليوم: شوفوا كم عدد أعضائكم الآن!
زيادة جديدة: محمد أضاف ابنه الجديد 👶
"
```

#### الأسبوع الثاني:
```
تحدي خاص:
"نقص الشجرة: ٥ أشخاص بدون صور
هل تقدرون تسدوا الفراغات؟"

سهل جداً: عشر الصور فقط، خمس دقائق
المكافأة: ٥٠ نقطة لكل شخص يساهم
```

#### الأسبوع الثالث:
```
عرض خاص (limited time):
"عودة العائلة الخاصة:

اشتروا النسخة المدفوعة بـ ٥٠% خصم
اختر: شهر مجاني أم خصم دائم
العرض ينتهي: يوم الجمعة"
```

### 8.2 علامات الإنذار المبكرة

قبل ما تنام عائلة، استشعر هذه الإشارات:

| الإشارة | التصرف | التوقيت |
|--------|---------|--------|
| لا توجد نشاطات لمدة ٧ أيام | إرسال "نشاط جديد من أحد أقاربك" | اليوم ٨ |
| عضو واحد فقط نشط | "ياسر وحده نشيط... أين الباقي؟" (humor) | اليوم ١٠ |
| لا توجد دعوات خارجية | "عائلتك لم تدع أحد جديد في أسبوع" | اليوم ١٤ |
| لم تضيف بيانات جديدة | "تحديث الآن: شوف من الجديد!" | اليوم ١٤ |

### 8.3 الإجراء النووي (Nuclear Option)

إذا ما استجابت العائلة بعد ٣ أسابيع:

```
"هل تريد أن تحذف حسابك؟ 😢

قبل ما تروح: بيانات عائلتك (٢٣ عضو) محفوظة
لو غيّرت رأيك، يمكن ترجع بدون مشاكل
فقط: قول لنا السبب؟

A) بطيء جداً
B) معقد
C) مو بالحاجة
D) سبب آخر"
```

**الفائدة:** بيانات قيمة عن سبب الفشل

## 9. الحافز الاقتصادي (Economic Retention)

### 9.1 النموذج المجاني vs المدفوع

| الميزة | مجاني | ساعد العائلة | احترف |
|-------|--------|------------|-------|
| عدد الأعضاء | ٣٠ | ١٠٠ | غير محدود |
| شارات | نعم | نعم | نعم + محرر |
| الإشعارات | basic | ذكية | VIP |
| تصدير PDF | لا | نعم | نعم + صور |
| الدعم | لا | بريد | ٢٤/٧ |
| السعر | مجاني | ٩٩ ريال/سنة | ٢٩٩ ريال/سنة |

### 9.2 "نموذج الترقية التدريجي"

```
المستخدم الجديد:
✓ يبدأ مجاني (٣٠ عضو)
✓ يضيف ٢ عضو جديد → ٥٠ عضو مجاني
✓ يدعو ٣ أعضاء من عائلة أخرى → ١٠٠ عضو مجاني
✓ أول حاجة يضربها: "ننك وصلت الحد الأقصى (١٠٠)"

👉 الخيار: ترقي أم احذف أقدم أعضاء
```

### 9.3 الشراء الاجتماعي (Social Purchase)

## 12. رسالة تلخيصية للمؤسس (عبدالله)

### لماذا هذه الاستراتيجية؟

يا عبدالله، أنت بدأت بعائلة واحدة وبنيت شيء جميل. الآن الخطوة التالية أصعب: **كيف توسع؟**

الحل ليس "نسويق أكتر" أو "ميديا أكتر". الحل أبسط:

**1. اجعل كل عضو راضٍ يصير سفير**
- زر "شارك" واضح جداً
- شهادات حقيقية من الأعضاء
- حافز عملي (نقاط، شارات، خصم)

**2. اجعل التطبيق عادة، مش كتاب**
- إشعارات شخصية (ليست عامة)
- فعاليات أسبوعية (تحديات، مسابقات)
- تجديد محتوى مستمر

**3. اقبض مقابل الخدمة العالية**
- المجاني لـ ٣٠ عضو (يكفي لعائلة صغيرة)
- الدفع لما يجاوز (سهل جداً ومنطقي)

**النتيجة؟**
- ستة أشهر: ٥–١٠ عائلات
- سنة: ٥٠ عائلة على الأقل
- كل عائلة = نمو عضوي جديد

كل هذا **بدون ميزانية تسويق كبيرة**. مجرد استراتيجية واضحة وتطبيق موثوق.

## الخلاصة

| الركيزة | الهدف | الأداة |
|--------|-------|--------|
| **الفيروسية** | كل عضو يدعو ٢ | زر شارك + تحديات |
| **الاحتفاظ** | يعود كل أسبوع | إشعارات ذكية + فعاليات |
| **Gamification** | يشعر بالإنجاز | نقاط + شارات + لوحة صدارة |
| **الاقتصاد** | يدفع لما يحتاج | نموذج freemium واضح |

**في النهاية، النمو الحقيقي يأتي من:**
- ✅ منتج جيد (عندك)
- ✅ مستخدمين سعداء (اجعلهم يشوفون القيمة)
- ✅ سهولة الدعوة (اجعل الدعوة أسهل من عدم الدعوة)
- ✅ حافز اقتصادي صحيح (لا تكون جشع، لكن احترم عملك)
